Section 0 · Scope & Correction

Scope & a Correction Up Front

This document is a working analysis prepared in response to a specific question raised by the Orbital founding team: are we building the right artifacts to maximise income from the Drips Stellar Wave Program, and what should be open versus closed source given the wave issues already drafted?

The first deliverable on this engagement included an unverified statistic — that the hintents reference repository contained "40 SCF references in docs/proposal.md." On direct inspection of the file the actual count is one sentence mentioning the Stellar Community Fund. The conclusion in the prior memo was therefore built on a confabulated correlation. This document supersedes that analysis and is based only on facts verified by reading the source files directly.

What that correction changes

The earlier hypothesis was: "featured repos are featured because they reference SCF/OnlyDust heavily." The verified picture is different — hintents has no FUNDING.json, no OnlyDust references, no Drips file, and exactly one SCF mention. Yet it still received $1,500. The real differentiators are shipped-product perception and documentation density, not explicit grant-program linkage. The corrected fix list at the end of this document reflects the verified evidence.

Section 1 · Drips Funding Pipeline

How Drips Actually Pays Stellar Projects

Drips itself is a neutral routing protocol. It does not decide which projects get funded — it splits incoming donations according to each recipient's "splits config." The interesting question is therefore "who sends money into Drips, and how do they pick recipients?"

The flow

Grant pool / SCF awardee / Wave-program treasury / direct sponsor
        │
        │  donates to a "Drip List" or to a project's Drips identity
        ▼
Drips protocol (smart contract)
        │
        │  splits per the recipient's configured splits graph
        ▼
Maintainer addresses + dependency graph + listed contributors

Money sources on Stellar (in order of materiality)

SourceHow it paysPredicts featured?
Stellar Wave Program A Drips-run initiative (the same Stellar Wave label used in WAVE_ISSUES.md) funded with grant money. Issues opted into the program pay contributors per complexity points when merged. Yes — direct.
OnlyDust A bounty platform that posts and pays contributor tasks; payouts can route through Drips. Strong second-order signal.
SCF awardees Grant recipients sometimes route a portion of their award through Drips to reward downstream contributors. Indirect — depends on the awardee.
Direct project sponsors Individual donors topping up a Drip List for a project they like. Marginal in volume.
Corrected mental model

Drips' "featured" status is curatorial, awarded to repos that look high-quality and shipped. Explicit SCF / OnlyDust integration is one validation path, but it is not necessary. Hintents is featured despite having no explicit OnlyDust or Drips integration anywhere in its repo — the curators picked it up on quality signal alone. That is the path Orbital should optimise for.

What that means for Orbital

Two paths to maximising Drips income operate in parallel and are not in tension:

  1. Be obviously high-quality and shipped. CHANGELOG with real releases, hosted docs, multi-language sophistication where relevant, an active contributor table. This is the look-the-part half.
  2. Activate the contributor flywheel. Publish the 150 wave issues, apply to OnlyDust, register on Drips. This is the turn-on-the-pipes half.
Section 2 · FUNDING.json

What FUNDING.json Contains and Means

This is a Drips-specific manifest distinct from GitHub's .github/FUNDING.yml (different schema, different purpose). It lives at the repository root.

Schema

{
  "drips": {
    "<chain-name>": {
      "ownedBy": "0x<ethereum-style-address>"
    }
  }
}

Real examples from the featured cohort

RepoChainAddress
be-energyoptimism0x7d012643D8E72994895F826A22E72ca972D576fd
stellar-portfolio-rebalancerethereum0xEFeA3500A667A359B84fc0ce4ba391F3B7761Bb8

What it does

  • Declares the canonical Drips recipient address for the repository, on a specific chain (Drips uses Ethereum-style addresses across multiple L2s).
  • When Drips' indexer crawls the repo, this file is the on-repo proof of ownership linking the GitHub repo to a Drips identity.

What it does not do

  • Does not unlock funding by itself. Being featured on Drips is a separate curatorial step; FUNDING.json just enables the routing once funding is sent.
  • Does not improve ranking. Of 15 featured repos audited in the Stellar cohort, only 2 have one. Hintents has none and still received $1,500.
  • Is not required. A maintainer can register repo ownership through Drips' web UI without the file. The file is a convenience and a proof, not a prerequisite.
When Orbital should add it

Only after registering an Orbital recipient on drips.network and obtaining the recipient address. Adding it before that is empty signaling — the file would point to nothing. Skip it for now; revisit after Drips registration is done.
